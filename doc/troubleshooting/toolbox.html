    Device Manager Troubleshooting

    Issue 1: HP Device Manager is non-functional for my device

    Solution:

    1. Run:
    $ hp-info
    1. Look at the output for:
    [ERROR]: Model query failed.
    1. First check and verify there are no connection issues with the printer.
    2. If the model query failed or there are errors in the output, then please report this to the support mailing list along with the hp-info output.

    Issue 2: When I click "Print..." in the toolbox, I receive an error.

    Solution:

    1. Install kprinter (part of KDE), or
    2. Install XPP

    Issue 3: When I click "Scan..." in the toolbox, I receive an error.

    Solution:

    1. Install xsane, or
    2. Install kooka

    Issue 4: The front panel display function always says "Display not available"

    Solutions:

    Generally only LaserJets and OfficeJets support this functionality. It is always turned off for DeskJets and other single or two function printers. If you feel that you are getting this message inappropriately, please submit this to the mailing list.

    Issue 5: I installed a printer in CUPS with a usb:, socket:, etc. backend and it doesn't show up in the Device Manager.

    Solution:

    Only devices installed that are using the "hp:" backend will display in the Device Manager. Re-install the printer using the "hp:" device URI and then re-fresh (with F6) or re-run the Device Manager.
